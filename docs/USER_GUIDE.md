## 1. 首次启动与登录

打开 app 后会进入登录页，点击 **「Google 登录」**，浏览器会弹出 Google 授权页面。

- 授权账号需有对应 Google Play Console 的 API 权限
- 登录成功后 token 会持久化到本地，不需要每次重新登录
- token 过期或权限不足时，app 会提示重新登录

## 2. 工作区概览

主界面分为三栏：

```
左栏（工作区）  →  中栏（功能选项）  →  右侧内容区
```

左栏有两个主工作区：

| 工作区 | 包含功能 |
|--------|---------|
| **Review** | 评论回复、批量回复、模板管理、知识配置、配置页 |
| **Tester** | Sheets 选择、Slides 选择、生成测试用例、对比用例、Gmail、App Script |

## 3. 评论回复（Review）

### 3.1 单应用拉取

1. 在中栏下拉框选择要处理的应用（包名）
2. 点击 **「拉取评论」**，拉取该 app 最近 7 天评论
3. 用页面上方的筛选栏按星级 / 回复状态 / 日期过滤列表

> 拉取结果会自动保存快照（`~/.tester-app/reviews-cache/`），下次进页面直接加载，不用重新拉。

### 3.2 批量拉取

1. 先在 **「配置 → Play Console 拉取配置」** 中启用各 app 并配好筛选条件（日期预设 / 星级 / 回复状态）
2. 回到评论页，点击 **「📦 批量拉取」**
3. 所有启用的 app 并行拉取，结果合并为一个列表（每条带应用标签）

### 3.3 AI 回复

点击评论卡片上的 **「🤖 AI 回复」**：

1. 输入回复方向（一句话，如"礼貌感谢并引导重新尝试"），可留空（模型自行判断）
2. 选择回复语言（默认跟随评论语言）
3. 等待生成 **3 条不同风格**的候选回复
4. 点选一条（或手动修改），字符计数会实时显示（上限 350）
5. 点 **「提交回复」**

> 可同时开多条 AI 回复弹窗，生成任务排队执行，缩小后会变成右下角的悬浮条。

### 3.4 模板回复

点击评论卡片上的 **「📋 模板回复」**：

1. 弹窗列出你收藏的模板（通用 + 该 app 专用，按 category 分组）
2. 点击某条模板，自动按评论语言匹配预存译文填入输入框
3. 可微调后提交

> 无对应语言译文时回退英文，并有提示。

### 3.5 分析评论

点击评论卡片上的 **「🔍 分析」**：

- 弹窗自动开始分析，注入该 app 的知识配置，输出：
  - 评论分类、问题点、信息缺口
  - 整体判断
  - 一条推荐回复（可直接调整后发送）
- 分析任务可缩小为悬浮条（左下角），支持同时分析多条

## 4. 批量回复（Batch Reply）

适合每天批量处理未回复评论。

### 4.1 配置（先做一次）

进入 **「配置 → Batch Reply 配置」**：

- 每个 app 一张卡片，可独立配置：
  - 日期预设（自上一个工作日 / 昨天 / 今天 / 近 7 天 / 自定义）
  - 目标星级
- 勾选「启用」的 app 才会被批量拉取
- 配置好后点 **「保存」**

### 4.2 执行

1. 进入 **「Batch Reply」** 页
2. 点击 **「拉取候选评论」** —— 并行拉取所有启用 app，按各自配置筛选
3. 评论按 app 分组展示
4. 点击 **「🔎 匹配模板并填充」**（顶部可选回复语言，默认 auto 跟随评论语言）
   - 命中模板的评论：自动填入翻译好的模板正文
   - 未命中的：标「未匹配·需手动处理」，可手填或点「🤖 AI 回复」现场生成
5. 检查/调整各条回复文案
6. 逐条点 **「提交」**，或点 **「一键提交全部」**

> **「✋ 人工处理」**：点此标记后，该评论会在下次「匹配模板并填充」中跳过，但仍可手填或提交。标记跨重启持久化。

> **费用提示**：匹配完成后工具栏会显示本次 token 用量和费用，实测约 $0.31 / 6 条评论。

## 5. 模板管理

进入 **「Review → 模板管理」**。

### 产品 Tab

- 模板按产品分组，每个 Tab 对应一个产品
- 点 **「管理关联」** 可编辑包名 ↔ 产品映射（新增/删除/修改）
- hover 产品 Tab 右上角出现 × 可删除整个产品（二步确认）

### 增删改模板

- **新增**：点顶部 **「+ 新增模板」**，填类别名和正文
- **编辑**：直接点模板行上的字段修改后保存
- **删除**：行内二步确认删除

### 收藏

- 每条模板左侧有 ★ 按钮
- 收藏的模板会出现在评论页的「📋 模板回复」弹窗中

### 批量导入 / 导出

- **导入 xlsx**：点 **「📥 从 xlsx 导入」**，选文件后确认（覆盖该产品现有模板）
- **导出 xlsx**：点导出按钮，下载当前产品的模板表格（A 列类别 / B 列英文 / C 列起各语言）

### 多语言预翻译

模板存英文或中文源文，可批量预翻译到其他语言（供模板回复按评论语言自动匹配）：

1. 选中需要翻译的模板
2. 选目标语言，点 **「补全多语言」**
3. 进度条 + 日志实时展示，可随时停止

## 6. 知识配置

进入 **「Review → 知识配置」**，按产品 Tab 编辑该 app 的知识块（Markdown 格式）。

- 知识块在「🔍 分析」时注入提示词，帮助模型理解 app 背景（功能说明、常见问题、已知限制等）
- 左侧编辑，右侧实时预览
- 写好后点 **「保存」**

## 7. Gmail 邮件

进入 **「Tester → Gmail」**。

### 首次配置

1. 部署 `docs/gmail-sync.gs` 脚本到 Gmail 账号（Apps Script 定时同步未读邮件到 Google Sheet）
2. 回到 Gmail 页，粘贴 Sheet 链接，配置：
   - Chrome profile（用哪个账号打开邮件）
   - 关联的邮件回复模板产品

### 日常使用

- 列表每封显示发件人 / 主题 / 机翻中文摘要
- 点 **「详情」** 看原文
- 点 **「↗」** 用指定 Chrome profile 打开邮件
- 点 **「已读」** 在本地隐藏（可用「↩ 撤销」找回最近一封）

## 8. 测试用例生成

### 8.1 选测试数据

**Sheets 页**：
- 左侧列表选 Google Sheet，或粘贴链接，或上传本地 xlsx（自动转 Google 表格）
- 右侧预览数据，切换 Tab 选择要用的 sheet

**Slides 页**：
- 选 Google Slides 后加载幻灯片
- 在缩略图条勾选要生成用例的页面（支持多选）

### 8.2 生成

1. 选好 Sheets 和 Slides 后进入 **「Generate」** 页
2. 页面展示当前选择摘要
3. 点 **「生成」** —— 导出 CSV + PPTX 后调用 Claude CLI
4. 终端风格日志实时显示进度
5. 生成完成后可 **「上传到 Drive」**，供后续对比使用

### 8.3 多轮对话

生成完成后，下方输入框可继续追问或要求修改，Claude 会基于同一会话继续响应。

## 9. 用例对比（Compare）

进入 **「Tester → Compare」**。

1. 左栏选「AI 生成版」Sheet（Drive 文件）
2. 右栏选「人工修改版」Sheet
3. 点 **「生成对比报告」**
   - 自动导出两份 HTML 并运行 diff 脚本
   - 日志实时输出
4. 点 **「在 Chrome 中打开」** 查看 HTML 报告（高亮新增 / 删除 / 修改的行）
5. 可点 **「反馈」** 把报告和源文件打包发给维护者（需已配置 Telegram）

## 10. 设置

进入右上角 **「Settings」**。

| 选项 | 说明 |
|------|------|
| **模型配置** | 分别为「回复 / 分析 / 翻译」选 Sonnet 或 Haiku（影响效果和费用） |
| **GitHub Token** | 填入后 app 可自动拉取私有 skill 更新 |
| **Skill 更新** | 查看各 skill 当前版本 / 最新版本，可手动触发更新 |
| **缓存管理** | 查看本地缓存大小，一键清理 |

## 常见问题

**Q：提示「需要重新登录」或「权限不足」**
A：重新点击登录，确认 Google 账号有 Play Console API 访问权限。

**Q：批量回复「匹配模板并填充」卡在运行中**
A：可点「停止」按钮中断。如长时间无日志输出，检查 Claude CLI 是否正常运行（`claude --version`）。

**Q：模板回复弹窗里没有模板**
A：先去「模板管理」给模板打 ★ 收藏，再回来就能看到了。

**Q：生成的多语言译文超过 350 字符**
A：翻译时会自动压缩重试一次；若仍超长，该条会标红警告，需手动缩短正文。

**Q：Skill 更新失败**
A：在设置页填入有效的 GitHub Token（需有对应私有仓库的 read 权限）。
